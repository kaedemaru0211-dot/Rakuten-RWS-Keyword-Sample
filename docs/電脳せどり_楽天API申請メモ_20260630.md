# 電脳せどり 楽天API申請メモ

作成日: 2026-06-30

## 1. この repo の使い方

このフォルダは、そのまま楽天API申請用の GitHub repo にする前提で整えてある。

- `README.md`: repo の説明文
- `index.html`: Application URL 用の公開ページ
- `scripts/search_keyword.py`: keyword 検索サンプル
- `.env.example`: APIキー設定例

先にやること:

1. GitHub に新規repoを作る
2. このフォルダを push する
3. `index.html` の `YOUR_NAME` と `your-email@example.com` を実値に変える
4. GitHub Pages を有効化する
5. 公開URLを楽天申請フォームの Application URL に入れる

## 3. 公開説明の方針

表に出すのは公式APIを使う検証コード。
スクレイピング前提の説明は出さない。

良い表現:

- 楽天市場商品検索APIを用いた商品検索・価格比較検証ツール
- keyword検索による商品情報取得サンプル

避ける表現:

- 楽天からデータを吸い上げる
- スクレイピングで楽天商品を収集する

## 4. せどりはスクレイピングでやるものか

結論: 公式APIがあるならそれを使う。スクレイピングは代替手段。

理由:

- ToS違反リスクを下げやすい
- サイト負荷を抑えやすい
- JAN、価格、URLなどを構造化で取りやすい
- 保守が軽い

この案件の方針:

- 楽天、Yahoo は公式API優先
- APIがない仕入れ先だけ必要に応じてスクレイピング
- スクレイピングは最小化

## 5. 注意

- 2026-06 時点の楽天市場商品検索APIは `applicationId` に加えて `accessKey` も必要
- `localhost` やダミーURLを Application URL に入れない
- 空repoのまま申請しない
